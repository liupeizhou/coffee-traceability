# ☕ 咖啡溯源系统 - 产品需求文档 (SPEC)

## 1. 产品概述

### 1.1 产品名称
咖啡全生命周期溯源系统 (Coffee Traceability Platform)

### 1.2 产品定位
一款覆盖咖啡从种植、采收、初加工、仓储、烘焙到杯测的全链路溯源管理平台，为产业链各环节参与者提供数据录入与可视化展示能力，同时为终端消费者提供扫码溯源的信任背书。

### 1.3 核心价值主张
- **透明可追溯**: 消费者扫码即可查看咖啡豆的完整生命周期数据
- **数据贯通**: 打通种植端、加工端、仓储端、烘焙端数据孤岛
- **角色权限**: 精细化权限管理，确保各角色只能操作对应环节数据

## 2. 目标用户与痛点分析

### 2.1 目标用户

| 角色 | 用户画像 | 核心需求 |
|------|----------|----------|
| **咖农 (Farmer)** | 云南/海南咖啡庄园主，50岁以下，智能手机用户 | 记录种植与采收数据，上传产地证明 |
| **加工厂 (Processor)** | 咖啡初加工企业，食品加工资质 | 记录处理法、发酵参数、品质检测数据 |
| **烘焙师 (Roaster)** | 精品烘焙坊主理人 | 记录烘焙曲线、杯测评分、风味描述 |
| **门店 (Cafe)** | 连锁/单体咖啡门店 | 展示溯源信息，增强顾客信任 |
| **农业局 (Government)** | 农业政策管理人员 | 全局数据统计与监管 |
| **终端消费者** | 精品咖啡爱好者 | 扫码溯源，了解咖啡身世 |

### 2.2 核心痛点

| 痛点 | 描述 | 解决方案 |
|------|------|----------|
| **信息不透明** | 消费者无法得知咖啡豆的真实来源与处理过程 | 全链路数据上链，扫码可见 |
| **数据孤岛** | 种植、加工、烘焙各环节数据割裂 | 统一批次号串联全生命周期 |
| **信任缺失** | 精品咖啡市场鱼龙混杂，缺乏可信背书 | 关键数据节点（采收、处理、杯测）可验证 |
| **监管困难** | 农业局难以掌握全域咖啡产业数据 | 统一数据面板，全局统计分析 |

## 3. 功能模块定义

### 3.1 核心功能模块

```
┌─────────────────────────────────────────────────────────────┐
│                    咖啡溯源系统                              │
├─────────────┬─────────────┬─────────────┬─────────────────┤
│   种植管理   │   加工管理   │   仓储管理   │    烘焙管理     │
│  Planting   │ Processing  │   Storage   │    Roasting    │
├─────────────┴─────────────┴─────────────┴─────────────────┤
│                     溯源查询 Traceability                    │
├─────────────────────────────────────────────────────────────┤
│                     用户权限管理 Auth                        │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 功能详情

#### 模块 A: 种植与采收 (Planting)
- [ ] 创建批次 (Batch) 并生成唯一批次号 (batchNumber)
- [ ] 记录种植地块、土壤数据、气候数据 (日照时长、昼夜温差、降雨量)
- [ ] 记录采收时间、采收量、果实品质
- [ ] 上传产地证明图片

#### 模块 B: 初加工与发酵 (Processing)
- [ ] 选择处理法 (水洗/日晒/蜜处理/二氧化碳浸渍/其他)
- [ ] 记录处理时长、发酵参数、pH值变化
- [ ] 记录特殊工艺备注
- [ ] 初加工完成后进入仓储环节

#### 模块 C: 仓储管理 (Storage)
- [ ] 记录仓储条件 (温湿度)
- [ ] 记录含水率、水活性、密度等物理参数
- [ ] 记录仓储时长

#### 模块 D: 烘焙与杯测 (Roasting)
- [ ] 记录烘焙设备、烘焙曲线
- [ ] 记录 Agtron 豆值/粉值
- [ ] 记录杯测评分 (SCAA 80分制)
- [ ] 记录风味描述 (风味轮标签)

#### 模块 E: 溯源查询 (Traceability)
- [ ] C端扫码查询入口 (基于 batchNumber)
- [ ] 时间线格式展示全生命周期数据
- [ ] 各环节数据卡片展示

#### 模块 F: 权限管理 (Auth)
- [ ] 用户注册与登录 (邮箱/手机号)
- [ ] 角色分配 (Admin/Government/Farmer/Processor/Roaster/Cafe)
- [ ] 基于角色的接口权限控制 (RBAC)

## 4. 非功能需求

### 4.1 性能要求
- 溯源查询接口响应时间 < 200ms (P99)
- 支持单批次百万级查询 (CDN 缓存)

### 4.2 安全要求
- 敏感数据 (个人身份信息) 需加密存储
- API 接口需身份验证 (JWT)
- 写操作需记录操作日志

### 4.3 可用性
- 系统可用性目标: 99.9%
- 溯源页面需支持 SEO (SSR)

## 5. 验收标准 (High-Level)

| 功能 | 验收条件 |
|------|----------|
| 批次创建 | 可创建批次并生成唯一 batchNumber (格式: CF-YYYYMMDD-XXXX) |
| 种植记录 | 可保存完整种植与采收数据 |
| 加工记录 | 可选择处理法并记录发酵参数 |
| 仓储记录 | 可记录物理参数 (含水率、水活性等) |
| 烘焙记录 | 可保存烘焙曲线、杯测评分、风味描述 |
| 溯源查询 | 扫码/输入 batchNumber 可展示完整时间线 |
| 权限控制 | 各角色只能操作对应模块数据 |

---

## 6. 边界约束

- **一期范围**: 仅支持咖啡豆原豆溯源，不涉及液态咖啡饮品
- **数据归属**: 各角色数据归各自所有，跨角色数据共享需授权
- **离线支持**: 暂不支持离线数据同步，需全程在线操作
